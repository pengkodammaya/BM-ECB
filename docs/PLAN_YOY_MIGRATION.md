# Plan: YoY as Primary Metric

**Date**: 2026-05-30  
**Status**: Pending  
**Priority**: High

---

## Problem

Currently, the main nowcast uses QoQ SA (quarter-over-quarter seasonally adjusted) as the primary metric, while DOSM reports YoY (year-over-year). This creates confusion:
- Dashboard shows mixed metrics
- Backcast is QoQ, actuals are YoY
- Leaderboard uses QoQ, DOSM advance estimate is YoY
- No direct comparison possible

## Goal

Switch to YoY as the single primary metric throughout:
- Main nowcast
- Backcast
- Leaderboard scoring
- Dashboard display
- Backtest evaluation

---

## Current State

| Component | Metric | Column |
|-----------|--------|--------|
| Main nowcast | QoQ SA | `dfm`, `bvar`, `beq`, `ensemble` |
| YoY nowcast | YoY | `dfm_yoy`, `bvar_yoy`, `ensemble_yoy` |
| Backcast | QoQ SA | `dfm_backcast`, `bvar_backcast`, `beq_backcast` |
| Actual (QoQ) | QoQ SA | `actual_gdp_pct` |
| Actual (YoY) | YoY | `actual_yoy_gdp` |
| Leaderboard | QoQ SA | `score_log` uses `"gdp_qoq"` |

## Target State

| Component | Metric | Column |
|-----------|--------|--------|
| Main nowcast | YoY | `dfm`, `bvar`, `beq`, `ensemble` |
| Backcast | YoY | `dfm_backcast`, `bvar_backcast`, `beq_backcast` |
| Actual | YoY | `actual` |
| Leaderboard | YoY | `score_log` uses `"gdp_yoy"` |
